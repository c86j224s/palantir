# Palantir K8s GUI - Phase 4 Postmortem (Generic Resource & Detail)

## 1. 중요한 점 (Key Takeaways)
* **범용성(Generality)의 힘**: 리소스별 개별 구현 대신 `DynamicObject`와 GVK를 활용한 범용 엔진을 구축하여 확장성을 확보했습니다.
* **데이터 접근성**: 복잡한 YAML에서 핵심 정보(환경 변수 등)를 추출해 전용 뷰로 제공하는 것이 실제 사용자 경험에 결정적입니다.

## 2. 잘된 점 (What Went Well)
* **입체적 검증**: 백엔드 통합 테스트와 프론트엔드 유닛 테스트를 병행하여 데이터 흐름 전반을 검증했습니다.
* **타입 정합성 해결**: 라이브러리 버전 충돌 문제를 근본적으로 파악하고 Cargo.toml 레벨에서 수정했습니다.

## 3. 잘못된 점 / 실수했던 부분들 (Mistakes & Pitfalls)
* **검증 프로세스 누락**: 빌드 성공 보고 전 프론트엔드 유닛 테스트를 생략한 점.
* **테스트 코드 설계 미흡**: 텍스트 매칭 오류 및 Mock 데이터 초기화 누락으로 인한 테스트 실패.
* **의존성 관리 소홀**: 필요한 라이브러리(kube 등)를 제때 추가하지 않아 빌드 에러 유발.

## 4. 개선할 점 (Future Actions)
* **'완료'의 정의 재정립**: 코드 작성 완료가 아닌, '백/프론트 테스트 통과 및 빌드 성공'을 완료의 기준으로 삼겠습니다.
* **선 질문 후 행동**: 환경이나 설계가 모호할 경우 임의로 추측하지 않고 사용자에게 질문하여 명확한 가이드를 확보하겠습니다.
